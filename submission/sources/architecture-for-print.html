محرك التجارة الإلكترونية عالي الأداء

مادة البرمجة المتوازية — Laravel + Octane (Swoole)

1. فكرة المشروع

API لتجارة إلكترونية يخدم عدداً كبيراً من المستخدمين المتزامنين. الهدف منع أخطاء المخزون (Race Condition)، عدم إرهاق السيرفر، وتأجيل المهام الثقيلة (فواتير، إشعارات) إلى الطوابير.

2. مخطط عام

  Client (Postman / JMeter)
           |
           v
  +---------------------------+
  | Octane (Swoole workers)   |
  | Middleware: سعة + circuit |
  +---------------------------+
           |
     +-----+-----+
     v           v
  Actions     Metrics (/metrics)
     |
  Services: Inventory (lock), Payment, Cache
     |
  MariaDB          Queue workers

4. AOP (نقاط المراقبة)

استخدمنا ConcurrencyAspect و Middleware لتسجيل مدة العمليات عند: قفل المخزون، الدفع المحاكى، وطلبات HTTP — دون تكرار نفس كود القياس في كل Action.
